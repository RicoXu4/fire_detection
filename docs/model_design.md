# 火灾检测模型设计与部署方案

## 模型目标

系统面向监控摄像头、巡检图片和云端图片接口的火焰识别任务。输出包括目标类别、置信度、检测框坐标和带框可视化图片。

## 模型结构

工程使用 YOLO v26 兼容权重作为检测模型入口，推理封装位于 `app/detector.py`。模型接口遵循 YOLO 单阶段目标检测流程：

1. 输入图像按 `imgsz` 缩放和归一化。
2. Backbone 提取火焰纹理、颜色、边缘和上下文特征。
3. Neck 融合多尺度特征，提升小火点和远距离火焰的检测能力。
4. Head 输出类别概率、目标置信度和边界框。
5. NMS 按 `conf` 和 `iou` 过滤重复框。

类别建议最少包含：

- `fire`: 明火、火舌、燃烧区域
- `smoke`: 烟雾，用于早期火灾辅助识别

如果部署场景只要求明火报警，可在 `FIRE_CLASSES=fire,flame` 中过滤烟雾类别。

## 数据提升策略

为了降低误报和漏报，训练集应覆盖以下场景：

- 白天、夜间、红外补光、低照度、逆光
- 室内、仓库、森林、道路、工厂、厨房
- 小目标远景、大面积火焰、遮挡火焰
- 难负样本：灯泡、车灯、夕阳、反光、红色广告牌、焊接火花

增强策略：

- HSV 色彩扰动，增强不同相机和光照的泛化能力
- Mosaic、随机缩放和平移，增强小目标识别
- 训练末期关闭 Mosaic，稳定边界框定位
- 按摄像头或场景划分验证集，避免同源图片泄漏导致指标虚高

## 评估指标

上线前至少检查：

- mAP50 / mAP50-95
- `fire` 类召回率，漏报成本高的场景应优先提高召回
- 每小时误报数，适合长期监控业务验收
- 小目标火焰、夜间火焰和难负样本的独立测试结果

## 部署路径

本工程提供两种部署方式：

- CLI 批处理：`scripts/infer.py`
- HTTP 服务：`app/api.py`

云端推荐流程：

1. 训练得到 `best.pt`，重命名或挂载为 `/models/yolov26_fire.pt`。
2. 使用 Docker 镜像部署 FastAPI 服务。
3. 根据硬件设置 `FIRE_DEVICE=cpu` 或 `FIRE_DEVICE=0`。
4. 根据业务风险设置 `FIRE_CONF_THRESHOLD`，高召回场景可降低到 `0.15-0.25`，低误报场景可提高到 `0.35-0.5`。
5. 通过 `/health` 做容器健康检查，通过 `/detect` 做图片识别。

## 输出格式

接口返回 JSON：

```json
{
  "image_width": 1920,
  "image_height": 1080,
  "detections": [
    {
      "label": "fire",
      "class_id": 0,
      "confidence": 0.91,
      "box": {"x1": 510.2, "y1": 220.6, "x2": 710.4, "y2": 530.1}
    }
  ],
  "visualization": "base64_png"
}
```

`visualization` 为可选字段，可通过 `visualize=false` 关闭以减少接口响应体大小。
